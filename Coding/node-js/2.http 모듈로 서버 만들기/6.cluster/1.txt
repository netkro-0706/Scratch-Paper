# cluster
코어가 8개여도 보통 코어 하나만 사용된다. 그걸 코어 하나당 노드 프로세스 하나를 배정시키는데 성능이 8배가 되는건 아니지만 개선된다.
기본적으로 싱글 스레드인 노드가 CPU 코어를 모두 사용할 수 있게 해주는 모듈
포트를 공유하는 노드 프로세스를 여러개 둘 수 있다.
요청이 많이 들어오면 병렬로 실행된 서버의 개수만큼 요청이 분산된다.
서버에 무리가 덜 간다.
단점으로 메모리, 세션등은 공유를 못 한다.

서버가 여러개 돌고 있을 때 유저가 1번 서버에 로그인 했다가 재접속하여 다른 서버에 가면 로그인이 풀린다. 이럴 땐 Redis 등 별도 서버로 해결한다.
