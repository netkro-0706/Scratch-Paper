1. pipe
스트림을 연결해서 한 스트림의 데이터를 다른 스트림으로 전달(복사) 할 때 사용하는 메서드
자동으로 데이터를 전달하며 메모리를 효율적으로 사용한다.

2. 파일 및 폴더 생성, 삭제
fs.access(경로, 옵션, 콜백) - 폴더나 파일에 접근 할 수 있는지를 확인
fs.existsSync - 파일 또는 폴더가 존재하는지 확인
fs.stats - 파일인지 폴더인지 확인
fs.mkdir(경로, 콜백) - 폴더를 만드는 메서드. 이미 폴더가 있으면 에러가 발생하므로 access()메서드로 확인을 하는것이 중요하다.
fs.open(경로, 옵션, 콜백) - 파일의 아이디를 가져오는 메서드. 파일이 없다면 파일을 생성 후 아이디를 가져온다.
가져온 아이디로 fs.read(), fs.write()로 읽거나 쓸 수 있다.
두번째 인자로 쓰기w,읽기r,기존 파일에 추가a 를 설정할 수 있다.
fs.rename(기존 경로, 새 경로, 콜백) - 파일의 이름을 바꾸는 메서스. 기존 파일 위치와 새로운 파일 위치를 적으면 된다.
fs.readdir(경로, 콜백) - 폴더 안의 내용물을 확인할 수 있다.
fs.unlink(경로, 콜백) - 파일을 지운다. 파일이 없다면 에러가 발생하므로 있는지 먼저 확인해야 한다.
fs.rmdir(경로, 콜백) - 폴더를 지운다. 폴더안에 파일이 있으면 에러가 발생하므로 내부 파일을 지우고 호출해야한다.
