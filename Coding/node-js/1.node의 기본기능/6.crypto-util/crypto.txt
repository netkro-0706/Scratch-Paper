# 단방향 암호화
복호화가 불가능한 암호화

## Hash기법
문자열을 고정된 길이의 다른 문자열로 바꾸는 방식
비밀번호  ->  해시함수  ->  다이제스트
       <-x-        <-

### Hash사용하기
1.createHash(알고리즘)을 설정
・ md5, sha1, sha256, sha512 등이 있다.
・ 알고리즘마다 취약점이 발견되어 사용하지 않는게 좋은 것들도 있다.
・ 현재는 sha512로 괜찮지만 취약점이 발견되면 더 강화된 알고리즘으로 바꾸는게 좋다.

2.update(문자열)

3.digest(인코딩)
・ base64, hex, latin1이 주로 사용되는데, base64가 결과 문자열이 가장 짧아 애용된다.

#### 보안
컴퓨터의 발달로 기존 암화화 알고리즘이 위협 받는중이다.
sha512가 취약해지면 sha3로 넘어가야한다.
현재는 pdkdf2, bcryptm scrypt 알고리즘으로 암호화가 되고 있다.
Node는 pdkdf2, scrypt가 지원된다.
